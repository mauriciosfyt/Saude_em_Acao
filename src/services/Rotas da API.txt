1. Módulo de Autenticação (/api/auth)
Responsável pelo login e recuperação de senha. Todas as rotas deste módulo são públicas.

Método | URL Final e Completa | Descrição da Funcionalidade
POST | http://34.205.11.57/api/auth/solicitar-token | Passo 1 do Login: Envia e-mail e senha. Se as credenciais estiverem corretas, um token de 5 dígitos é enviado para o e-mail do usuário.
POST | http://34.205.11.57/api/auth/login-token | Passo 2 do Login: Envia o token recebido por e-mail para trocá-lo por um token JWT.
POST | http://34.205.11.57/api/auth/esqueci-senha/solicitar | Envia um e-mail para solicitar um token de redefinição de senha.
POST | http://34.205.11.57/api/auth/esqueci-senha/confirmar | Confirma a redefinição de senha usando o token recebido e uma nova senha.

2. Módulo de Gerenciamento de Usuários (/api)
Contém rotas para gerenciar Alunos, Professores, Admins e ações do próprio usuário logado.

2.1 Rotas para Alunos
POST | http://34.205.11.57/api/aluno | Pública | Cria um novo usuário com o perfil de ALUNO.
GET | http://34.205.11.57/api/aluno | Admin | Lista todos os alunos de forma paginada (filtro opcional ?nome=...).
PUT | http://34.205.11.57/api/aluno/{id} | Admin | Atualiza os dados de um aluno específico pelo seu ID.
DELETE | http://34.205.11.57/api/aluno/{id} | Admin | Exclui um aluno específico pelo seu ID.

2.2 Rotas para Professores
POST | http://34.205.11.57/api/professor | Admin | Cria um novo usuário com o perfil de PROFESSOR.
GET | http://34.205.11.57/api/professor | Admin | Lista todos os professores de forma paginada (filtro opcional ?nome=...).
PUT | http://34.205.11.57/api/professor/{id} | Admin | Atualiza os dados de um professor específico pelo seu ID.
DELETE | http://34.205.11.57/api/professor/{id} | Admin | Exclui um professor específico pelo seu ID.

2.3 Rotas para Admins
POST | http://34.205.11.57/api/admin | Admin | Cria um novo usuário com o perfil de ADMIN.
GET | http://34.205.11.57/api/admin | Admin | Lista todos os administradores de forma paginada.
PUT | http://34.205.11.57/api/admin/{id} | Admin | Atualiza os dados de um administrador específico pelo seu ID.
DELETE | http://34.205.11.57/api/admin/{id} | Admin | Exclui um administrador específico pelo seu ID.

2.4 Rotas de Autoatendimento do Usuário Logado
GET | http://34.205.11.57/api/meu-perfil | Qualquer Usuário Logado | Retorna os dados completos do perfil do usuário que está logado.
DELETE | http://34.205.11.57/api/me | Qualquer Usuário Logado | Permite que o usuário logado exclua sua própria conta.

3. Módulo de Produtos (/api/produtos)
Gerenciamento do catálogo de produtos da academia.

GET | http://34.205.11.57/api/produtos | Pública | Lista todos os produtos (filtro opcional ?nome=...).
GET | http://34.205.11.57/api/produtos/destaques | Pública | Retorna uma lista dos produtos mais reservados (ranking de popularidade).
GET | http://34.205.11.57/api/produtos/{id} | Pública | Retorna os detalhes de um único produto pelo seu ID.
POST | http://34.205.11.57/api/produtos | Admin | Cria um novo produto no catálogo.
PUT | http://34.205.11.57/api/produtos/{id} | Admin | Atualiza os dados de um produto existente.
DELETE | http://34.205.11.57/api/produtos/{id} | Admin | Remove um produto do catálogo.

4. Módulo de Reservas (/api/reservas)
Fluxo de solicitação e gerenciamento de reservas de produtos.

POST | http://34.205.11.57/api/reservas | Aluno | Um aluno logado solicita a reserva de um produto.
GET | http://34.205.11.57/api/minhas | Aluno | Retorna a lista de todas as reservas feitas pelo aluno logado.
GET | http://34.205.11.57/api/reservas | Admin | Lista todas as reservas do sistema (filtro opcional ?status=...).
GET | http://34.205.11.57/api/reservas/stats | Admin | Retorna estatísticas sobre as reservas (pendentes, aprovadas, etc.).
PATCH | http://34.205.11.57/api/reservas/{id}/aprovar | Admin | Altera o status de uma reserva para "APROVADA".
PATCH | http://34.205.11.57/api/reservas/{id}/rejeitar | Admin | Altera o status de uma reserva para "REJEITADA".
PATCH | http://34.205.11.57/api/reservas/{id}/cancelar | Aluno ou Admin | Cancela uma reserva (o aluno só pode cancelar as suas).

5. Módulo de Treinos (/api/treinos)
Gerenciamento de planos de treino e acompanhamento de progresso.

POST | http://34.205.11.57/api/treinos | Professor ou Admin | Cria um novo plano de treino.
GET | http://34.205.11.57/api/treinos | Pública | Lista todos os treinos (filtros opcionais ?nome=..., ?tipo=..., ?responsavelId=...).
GET | http://34.205.11.57/api/treinos/{id} | Pública | Retorna os detalhes de um plano de treino específico.
PUT | http://34.205.11.57/api/treinos/{id} | Professor ou Admin | Atualiza um plano de treino existente.
DELETE | http://34.205.11.57/api/treinos/{id} | Professor ou Admin | Exclui um plano de treino.
POST | http://34.205.11.57/api/treinos/{id}/realizar | Aluno | O aluno logado registra que concluiu um treino.
GET | http://34.205.11.57/api/treinos/desempenho-semanal | Aluno | Retorna a lista de treinos que o aluno logado realizou na semana atual.

6. Módulo de Dashboard e Chat
GET | http://34.205.11.57/api/dashboard/stats | Admin | Retorna estatísticas consolidadas (filtro opcional ?ano=...).
GET | http://34.205.11.57/api/chat/historico/{chatId} | Qualquer Usuário Logado | Retorna o histórico de mensagens de uma sala de chat.
